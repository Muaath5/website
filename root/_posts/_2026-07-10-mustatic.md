## Intro
Here is a documentation for an architecture that is benifical project ideas, for example:
- Hackathon Participation proof-of-concept (PoC)
- Personal/Work Portfolio
- Public data viewer website

It is prefered before you continue reading to know about these:
- Static Engine Generators: Jekyll, Liquid, Blade
- Github workflows

## History
It began in [informatics-sa/Website](https://github.com/informatics-sa/Website) where it was since [`01 May 2024`](https://github.com/informatics-sa/Website/commit/cb513788ccf164435f59938dc7eec8794105c0a1) a normal HTML/Jekyll with `_layouts` and I was writing them manually.
In [`15 Aug 2024`](https://github.com/informatics-sa/Website/commit/d03541372895c78d5e6fc4bf589b78704c44ed19) I wrote a `build.py` to make the process shorter for me.
After 3 days at [`18 Aug 2024`](https://github.com/informatics-sa/Website/commit/ca7370fbefc2053e0aa6a04f0bb96574bd71d5ce) I added a workflow that automates running `build.py` after each commit.
Skip to [`07 Sep 2024`](https://github.com/informatics-sa/Website/commit/ad93289be09b336b686dfa74ebc0b03aae3b0bd5) I cleared the root of the repo and used `root/` directory.
Later in [`13 Mar 2025`](https://github.com/informatics-sa/Website/commit/ddd1b03f5f7c435c5cd96ebb7dd4829173cd39d1) AliMark71 added a library that should be used by `build.py` and it reduced repeated code (Reading JSON file, Writing Frontmatter, etc.).
1 Day after in [`14 Mar 2025`](https://github.com/informatics-sa/Website/commit/59403ce7cbb292296db186065b6f3ce8eddac444) We did a huge improvement on i18n by adding `translations.json` (Check addons).
In [`04 Oct 2025`](https://github.com/informatics-sa/Website/commit/7817e5d3b6a8091cc571fcfeaf9a1d3ac928e3dc) we used `_layout/` folder as a submodule (Check addons).
In [`18 May 2026`](https://github.com/informatics-sa/Website/commit/b1396e071ac2fc5e2ced476863c3e89010a9d5c1) AliMark71 added secret database (Check addons).

## Overview
The three pillars:
- Data: JSON/Sqlite/CSV
- Builder: Python/Swift/Go
- Engine: Jekyll/Liquid

## Structure
Here is the best structure to be followed:
- `.github/workflows/pages-deploy.yml`
- `builder/`
  - `lib/`
  - `build.py`
- `data/`
  - `*.json`
  - `*.csv`
- `root/`
  - `_layouts/`
  - `*.html`
  - `*.md`

Although the examples use this structure, which is not the best:
- `.github/workflows/`
- `root/`
  - `data/`
  - `_layouts/`
- `build.py`

## Pros & Cons
### Pros
- Serverless: You can code projects with budget absolute 0$
- Fast Edit: A single commit writing some JSON in your mobile would change the whole website in minutes
- Static + Database: You could have an easy access to website's data even if they are sensitive
- Traffic Analysis: You can link it with Google Analytics and achieve statistics without having a database
### Cons
- Readonly database: you cannot write to a database 
- Manual initiating: currently it doesnt have many documentations so most of the work must be done manually
